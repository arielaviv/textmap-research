TextMap vs GeoJSON: how LLMs read maps

Preliminary results · real OpenStreetMap scenes · answers graded automatically against geometric ground truth · temperature 0 · each format shown in isolation

Setup. The same maps are given to the model in two representations — GeoJSON (raw geometry, the baseline an LLM agent normally receives) and TextMap (an ASCII grid + a measurement legend). The model answers 10 spatial questions per map; a geometry oracle grades every answer. Tokens, latency and hallucinated entity ids are logged per answer.

2. Scaling the model does not help GeoJSON

The same 20 maps, three model tiers. On raw GeoJSON, accuracy is flat — a stronger model extracts nothing more from raw coordinates. On TextMap, accuracy climbs with the model.

6. Not yet proven — next runs

  • One neighbourhood: all maps are distinct blocks within a ~2 km area of Manhattan (one street morphology). A second, differently-shaped city is next.
  • Held-out questions: the question set was designed alongside the format; a separate question set the design never saw will rule out benchmark tuning.
  • Anthropic models only so far; GPT / Gemini / open models are queued.
  • Sonnet/Opus points are 20 maps each (the 60-map run is the certified headline).

Reliability note: both text formats almost never invent entities (hallucinated-id rate ≤1%). Rendering the same maps to an image raised it to ~40% — rasterization destroys entity identity. Full four-format comparison (incl. WKT, image) to follow.
